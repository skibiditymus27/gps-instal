## Security Policy

### Supported Versions
- `main` branch
- Published release tags (if any)

### Reporting a Vulnerability
Please email the maintainer at `skibiditymus27` with:
- A concise description of the issue and impact
- Steps to reproduce or proof-of-concept
- Suggested remediation (if known)

You will receive acknowledgment within **5 business days**. Do not open public issues for security reports.

### Responsible Use
- Do not weaponize or repurpose this project for personal gain, harassment, or abuse.
- Follow applicable laws and obtain proper authorization before security testing any deployed instance.

### Disclosure Process
- We will validate, prioritize, and fix confirmed issues.
- You will be notified when a fix is available; coordinated disclosure is preferred.
